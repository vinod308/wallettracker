1 What is WalletTracker?

The core purpose, problem solved, and who uses it

The Problem It Solves

Garage Collective manages multiple clients across services like Digital Marketing, SEO, Performance Marketing, Web Development, Design, and more. Before WalletTracker, there was no single place to:

  • See how much each client is paying every month
  • Know which clients are at risk of churning
  • Identify which services a client is NOT using (upsell gaps)
  • Track contract renewal dates
  • Understand how much of a client's total marketing budget we capture (Wallet Share)
Definition — Wallet Share
Wallet Share is the percentage of a client's total marketing/service budget that Garage Collective captures. For example, if a client spends ₹10 lakhs/month on all marketing and pays us ₹2 lakhs/month, our wallet share is 20%. The goal is to increase this percentage by offering more relevant services.

What WalletTracker Does

WalletTracker is a full-stack web application that:

  1. Reads live data from Google Sheets — our existing invoice spreadsheet becomes the data source
  2. Processes and analyses it automatically — calculates MRR, growth rates, at-risk flags, service gaps
  3. Displays interactive dashboards — charts, tables, KPI cards, health scores
  4. Generates exportable reports — PDF, Excel, CSV for meetings and presentations
  5. Identifies upsell opportunities — tells you exactly which services to pitch to which client

2 How the System Works — Architecture

Technology stack, components, and how they connect

Three-Layer Architecture

Layer 1 — Frontend (What You See)

Built with React 18 + Vite + TailwindCSS. This is the browser interface at wallet.garageaistack.com. It includes 8 pages, interactive charts (using Recharts), and a responsive design that works on mobile and desktop.

Layer 2 — Backend (The Server Brain)

Built with Node.js + Express. Runs on the Digital Ocean server at port 5000. Handles:

  • User authentication (login, logout, password reset)
  • Fetching CSV data from Google Sheets and caching it for 5 minutes
  • API endpoints for profile, notifications, and user management
  • Security middleware (rate limiting, CORS, JWT tokens)

Layer 3 — Database (Permanent Storage)

Built with PostgreSQL. Stores user accounts, roles, session data, notification preferences, and audit logs. Client revenue data comes from Google Sheets, NOT the database.

How Pages Are Structured

All pages follow the same layout:

  • Header — Logo (desktop), notifications bell, profile dropdown
  • Sidebar — Navigation links to all 7 pages (slide-out on mobile)

Step 2 — Backend Proxy (5-Minute Cache)

The backend server fetches the Google Sheet as a CSV file and caches it for 5 minutes. This means:

  • Multiple users loading the dashboard simultaneously only trigger ONE Google Sheets fetch
  • Response time for users is near-instant (milliseconds instead of seconds)
  • Google Sheets rate limits are not hit
  • To refresh data: wait 5 minutes, or manually clear the cache

Step 3 — CSV Parsing & Column Normalisation

The frontend parser (PapaParse library) converts raw CSV text into structured JavaScript objects. Important fix applied: all column headers are trimmed of extra spaces (e.g. "Services " → "Services") to ensure consistent field access.

Step 4 — Analytics Engine (useCSVData Hook)

This is the brain of the application. From the raw parsed data it computes:

  • Client profiles — merges same client across multiple months
  • MRR calculations — service MRR + addon MRR = total MRR
  • At-Risk detection — if a client's revenue dropped ≥15% vs. previous month → flagged At Risk
  • Service detection — reads the Services and Addons columns and auto-detects service categories using keyword patterns
  • Upsell opportunities — compares detected services vs. all possible services to find gaps
  • Wallet Intelligence metrics — health score, growth rates, retention %, revenue share

Step 5 — Adding a New Month

How to Add August 2025 Data
1. Create a new Google Sheet tab named "august" with the same column headers
2. Open the backend file: backend/src/routes/sheets.routes.js — add the sheet ID for august
3. Open the frontend file: frontend/src/utils/csvParser.js — add august: { path: '/api/sheets/csv/august', month: 'August 2025', monthKey: 'august', order: 4 }
4. Rebuild and deploy — August data appears automatically across all pages

Indian Currency Format Handling

The system handles Indian number format (lakhs, crores) correctly:

  • 3,71,700 → ₹3,71,700 (three lakh seventy-one thousand seven hundred)
  • 2,94,995.28 → ₹2,94,995.28 (with paise)
  • 1,88.800 → ₹1,88,800 (auto-corrects spreadsheet typos where period is used instead of comma)
  • Fields with "+" separator (multiple amounts) → summed automatically

Network & API Security

  • HTTPS enforced — all traffic to wallet.garageaistack.com is encrypted via SSL/TLS
  • CORS protection — only requests from wallet.garageaistack.com are accepted by the backend
  • Rate limiting — maximum 5 requests per 15 minutes on sensitive endpoints (login, password reset) to prevent brute-force attacks
  • Helmet.js — sets 14 security HTTP headers automatically (prevents clickjacking, MIME sniffing, etc.)
  • SQL injection prevention — all database queries use parameterised statements, never raw string concatenation
  • Input validation — all form inputs validated server-side using express-validator

Google Sheets Data Security

  • The Google Sheets URL is stored only in the backend environment file (not in frontend code)
  • The frontend never directly contacts Google — all requests go through the backend proxy
  • The backend .env file is excluded from GitHub via .gitignore — credentials never pushed to source control
  • Cache is in-memory only — data is not written to disk
Important — What Data Is Stored in the Database
The PostgreSQL database stores ONLY user account data (names, emails, hashed passwords, roles, preferences). All client revenue data comes exclusively from Google Sheets at request time. No client financial data is permanently stored in the database.

Audit Logs

The system maintains audit logs for all significant actions: logins, password changes, profile updates, and user management actions. These logs record the user, timestamp, IP address, and action taken — useful for compliance and security reviews.

At Risk Detection Logic

How a Client Gets Flagged "At Risk"
The system compares a client's most recent month MRR vs. the previous month MRR. If revenue dropped by 15% or more, the client is automatically flagged as "At Risk". For example: if Carrier Airconditioning paid ₹2,00,000 in June and only ₹1,65,000 in July (a 17.5% drop), they become At Risk. This triggers a red badge and appears in the At Risk KPI card and AI insights.

Service Auto-Detection

The system reads the "Services" and "Addons" columns in the Google Sheet and uses keyword matching to detect which service categories apply. Detected services appear as tags on the client. This is used throughout the app for upsell analysis.

8.3 Wallet Share — What it Means

Understanding Wallet Share

Wallet Share = (What client pays Garage) ÷ (Client's total marketing budget) × 100

Example: If VIP Industries spends ₹20 lakhs/month on all marketing and pays Garage ₹3.7 lakhs, our wallet share is 18.5%.

The goal of WalletTracker is to grow this percentage by identifying and pitching the services the client currently buys elsewhere.

Contract Table

Each row shows: Client Name, Start Date, Estimated End Date, Days Until Expiry, Monthly Value (MRR), and Renewal Status. Rows are colour-coded by urgency:

  • Red — expires in 0–30 days — immediate action needed
  • Amber — expires in 31–60 days — start renewal conversation
  • Green — expires in 61–90 days — plan ahead

How Contract Dates Are Calculated

Since Google Sheets contains the "Contract Start" field, the system adds 12 months to calculate the estimated end date. For example, if Contract Start = "1 April 2025", the estimated end date = "31 March 2026". Days remaining = end date − today's date.

Best Practice — Contract Renewal Timeline
Start renewal conversations at 90 days before expiry. Send a formal renewal proposal at 60 days. Confirm and sign at 30 days. Contracts renewed before expiry maintain continuous revenue — a gap in renewal means a gap in invoicing.

Estimated Revenue Gain Calculation

Each upsell opportunity shows an "Estimated Gain" — calculated as: 20% of the client's current average MRR per recommended service. This is a conservative estimate based on typical service pricing relative to existing spend.

How to Use Upsell Data in Client Meetings

  1. Before the meeting — open the client's Detail page. Note their detected services and which ones are missing.
  2. Review the upsell card — on Dashboard and Client Detail pages, the top recommended services are shown with estimated revenue gain.
  3. Prepare a pitch — use the missing services as the agenda: "We currently handle your Digital Marketing. We've seen great results adding SEO and Content Marketing for similar clients in your industry."
  4. Reference the data — you can show the client the revenue trend chart demonstrating the value you've already delivered, then propose adding more services.
  5. After the meeting — if a new service is agreed, it will appear in next month's Google Sheet and automatically update all charts and analyses.

14 How to Increase Wallet Share

Strategic framework for capturing more of each client's total marketing budget

What "Increasing Wallet Share" Means

If a client's total monthly marketing budget is ₹15 lakhs and we invoice ₹3 lakhs, our wallet share is 20%. Every additional service we add — SEO, social media, paid ads, video — increases this share. The goal is to become the primary agency partner, not just one of many vendors.

The 5 Wallet Share Growth Strategies

Strategy 1 — Service Consolidation ("One Agency" Model)

Position Garage as the one-stop partner. Many clients use 3–4 different agencies (one for SEO, one for social, one for design). The pitch: consolidate everything with Garage for better coordination, unified reporting, and typically better pricing per service.

WalletTracker Supports This With
The "Missing Services" list on each client's upsell card shows exactly which services they likely use elsewhere — these are the consolidation targets.

Strategy 2 — Add-on Expansion

Clients who already buy add-ons are the best candidates for more add-ons. They have proven they are comfortable expanding the relationship. The add-on clients list on Wallet Intelligence shows who to target first.

Strategy 3 — Performance-Based Expansion

When a service performs well, use the data to justify adding more. Example: "Your Performance Marketing spend generated X leads at ₹Y per lead. Let's increase the budget and add Content Marketing to nurture those leads."

Strategy 4 — Retainer Lock-In with Incentive

Propose a consolidated retainer that bundles multiple services at a slight discount vs. individual pricing. Client gets value (lower per-service cost), Garage gets higher total MRR and better retention.

Strategy 5 — At-Risk Rescue = Retention + Upsell

When a client's revenue is declining (At Risk flag), do NOT immediately pitch more services. First, address the reason for decline. Once the relationship is stabilised, introduce a service that directly addresses their pain point — this can actually increase wallet share during a recovery phase.

How to Add a New Month (August onwards)

  1. Create the August 2025 Google Sheet with identical column headers
  2. Add the Google Sheets share link to the backend environment file
  3. Add one line to csvParser.js: august: { path: '/api/sheets/csv/august', month: 'August 2025', monthKey: 'august', order: 4 }
  4. Add the route in sheets.routes.js backend file
  5. Update useCSVData.js — add august to the MONTH_MAP and FULL_MONTH_ORDER
  6. Deploy — August data auto-appears across all pages, charts, and analytics
